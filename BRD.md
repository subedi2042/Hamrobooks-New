2) Business Requirements (BRD List)
Business Objectives
BO-01 (Access): Make Nepali books easy to find and purchase in the USA
BO-02 (Trust/Culture): Build a culturally grounded, trustworthy experience
BO-03 (Ops): Enable efficient bookstore operations (catalog → orders → fulfillment)
BO-04 (Growth): Improve discoverability, repeat customers, and scalability
Business Requirements
Storefront & Discovery
BR-01: Users can browse Nepali literature by genre, author, language, and collections
BR-02: Users can search by Nepali/English title, author, and transliteration
BR-03: Website must be mobile-first and optimized for fast browsing
Checkout & Fulfillment
BR-04: Users can purchase using a simple, secure checkout
BR-05: Customers can see US shipping options, costs, and delivery estimates
BR-06: Customers can view and track order status (email confirmation minimum)
Trust, Authenticity, Community
BR-07: Product pages must show ISBN/publisher/edition, descriptions, and related books
BR-08: Website must include an About/Mission page to communicate cultural purpose
BR-09: Website supports content publishing (blog/book reviews/author spotlights)
Admin & Management
BR-10: Admin can manage products, pricing, inventory, and images
BR-11: Admin can manage orders, refunds/returns (policy + workflow), and customer inquiries
BR-12: Admin can create promotions (discount codes, featured collections, banners)
Growth & Retention
BR-13: Website supports SEO fundamentals (metadata, friendly URLs, sitemap)
BR-14: Website includes analytics tracking (traffic, conversion, best sellers)
BR-15: Website supports email capture and newsletter integration
3) Stakeholders → Needs → Requirements Mapping
Primary Stakeholders
S-01: Customer (Nepalese diaspora reader)
S-02: Store Owner/Admin
S-03: Fulfillment/Shipping partner (could be owner initially)
S-04: Content editor (optional)
S-05: Payment provider / platform
Stakeholder Needs
SN-01 (Customer): Find books quickly, trust authenticity, easy checkout
SN-02 (Admin): Add/update products easily, manage orders efficiently
SN-03 (Shipping): Clear shipping rules, packaging workflow, labels
SN-04 (Content): Publish community content to grow engagement
SN-05 (Payments): Secure payments and compliance
4) Traceability: Business → Functional → Non-Functional (Mini Matrix)
Functional Requirements (FR)
FR-01: Browse catalog by categories/collections
FR-02: Filter by author, genre, language, price, popularity, new arrivals
FR-03: Search with partial matches; support transliteration keywords
FR-04: Product details page with ISBN/publisher/edition + synopsis
FR-05: Cart management (add/remove/update quantities)
FR-06: Checkout flow (guest checkout + optional account)
FR-07: Payment processing integration
FR-08: Shipping calculation rules (US-only or US + Canada if desired)
FR-09: Order confirmation email + order history page (if account)
FR-10: Admin portal for products/inventory/orders/promotions
FR-11: Content pages: About, Blog, Community, Contact form
FR-12: Newsletter signup integration
Non-Functional Requirements (NFR)
NFR-01 Performance: pages load under ~2–3 seconds on mobile
NFR-02 Security: HTTPS, secure payment handling, no card data stored on site
NFR-03 Availability: target uptime aligned to hosting/platform (e.g., 99.9%)
NFR-04 Accessibility: WCAG-aligned basics (contrast, keyboard nav, alt text)
NFR-05 SEO: structured metadata, clean URLs, sitemap, indexable pages
NFR-06 Privacy/Compliance: privacy policy, cookie notice (as needed), data handling
NFR-07 Scalability: can expand to 5k–20k SKUs over time without redesign
